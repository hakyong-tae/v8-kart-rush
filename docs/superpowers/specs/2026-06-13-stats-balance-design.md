# 캐릭터·카트 스탯 밸런스 설계 (2026-06-13)

MK8 Deluxe 능력치 체계(히트맵 배치표 + Weight Classes)를 레퍼런스로 한 7스탯 모델.
유저와 브레인스토밍으로 확정 (위젯 히트맵 3회 이터레이션).

## 모델

- **보이는 4스탯** (UI 4칸 막대, 1~10 / 6=중립): 속도 · 가속 · 핸들 · 드리프트
- **숨은 3스탯** (차체 크기·플레이 느낌으로 짐작, `<i class="ti ti-lock">`): 무게 · 미니터보 · 무적
- **무게 클래스**: Light / Medium / Heavy — MK8 대각선 (Light=민첩·터보 ↔ Heavy=무게·무적)

## 밸런스 규칙 (확정)

1. **카트: 보이는 4스탯 합 = 24 전 카트 고정** — UI에서 우열 없이 공정하게 보임.
   클래스 개성은 숨은 스탯 전담. 숨은 합은 클래스별 차이 허용(MK8식): Light 11~14 / Medium 15~17 / Heavy 17~20.
2. **캐릭터: 보이는 4합 = 0, 숨은 3합 = +2** — 전원 총합 +2 황벨.
   모카(무게+1·무적+1) 코코(드리프트+2·터보+2·속도가속-1) 피코(핸들+2·무적+2) 라임(속도+2·무게+1·터보+1) 토토(가속+2·터보+1·무적+1).
3. **박시 B 방어 중첩 완화**: 무적 9→7 (아이템전 무게10+무적9 중첩 OP 방지 — 판단 근거는 대화 로그).
4. 총합 스프레드(카트 35~44)는 모드별 가치 분산으로 용인: 무게·무적은 아이템전 화폐, 미니터보는 스피드전 화폐.

## 물리 환산 (`pointsToMultipliers`)

| 스탯 | 환산 | 반영 위치 |
|---|---|---|
| 속도 | 1 ± 2.5%/pt | MAX/BOOST/BOOSTER_SPEED 배율 |
| 가속 | 1 ± 2.5%/pt | ACCEL 배율 |
| 핸들 | 1 ± 2.5%/pt | GRIP_NORMAL·GRIP_ICE (일반 주행 그립) |
| 드리프트 | 1 ± 2.5%/pt | GRIP_DRIFT·GRIP_DRIFT_ICE (드리프트 그립) |
| 미니터보 | 게이지 ±5%/pt · 릴리스 부스트 지속 ±6%/pt | GAUGE_RATE, 티어 릴리스 applyBoost |
| 무게 | 질량 1 ± 0.12/pt | resolveKartCollision — 밀림/감속을 질량비로 배분 |
| 무적 | 스핀시간 ∓5%/pt · 회복 후 무적 0.5s ± 0.1s/pt | SPIN_TIME 배율, hitGraceT (연속피격 방지) |
